notes for the maintainer

  • Every glyph is real: ✓ ✗ □ › ● — all renderable with lipgloss. The spinner is the same braille cycle already in styles.go.
  • Hardware chrome around the terminal is for marketing/docs only — the actual TUI stops at the screen. Don't ship the bezel inside Bubble Tea.
  • Orange stays reserved for ACTIVE. Success toast is dust-green so the eye tracks failures first. Keep the rule in styles.go.
  • Press-with-args (station 04) is new — current TUI bails to CLI for required args. The arg form is a single-column bubbletea huh?-style prompt with Tab/Enter semantics.
  • Detail view (station 05) mirrors cmd/detail.go output but reorganized as a TUI page — same data, better hierarchy.
  • Paper & phosphor themes: gate on BUTTONS_THEME env (phosphor, paper, default). Current BuildStyles() only branches on dark/light — extend it.
