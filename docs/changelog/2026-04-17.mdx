## Agent skill picker in buttons init

`buttons init` now sets up your coding agent to use Buttons automatically.

### New features

- **Agent skill picker** — when you run `buttons init` on an interactive terminal, a picker asks which coding-agent skill files to install. Supported agents: Cursor, Claude Code, Cline, GitHub Copilot, and AGENTS.md. Nothing is installed unless you explicitly select it. Use `--agent cursor,claude` to skip the picker, or `--agent none` to opt out entirely. See the [CLI reference](/cli/buttons_init).

- **`.buttons/AGENT.md` reference file** — `buttons init` now writes a short reference doc at `.buttons/AGENT.md` that teaches any coding agent what Buttons is and how to use it. The file is always kept up to date on re-runs. See [folder structure](/concepts/folder-structure).

### Updates

- **Idempotent skill updates** — re-running `buttons init` on a project that already has skill files replaces the Buttons section in place (using start/end markers) instead of duplicating content.

### Bug fixes

- **Agent picker defaults to single-select** — the interactive agent picker now uses a single-select list instead of a multi-select. Previously, pressing Enter without first pressing Space submitted an empty selection, which silently skipped installation. The picker also includes an explicit "None (skip)" option so opting out is a clear, intentional choice. You can still install multiple agent skills at once with `--agent cursor,claude`. See the [CLI reference](/cli/buttons_init).
</Update>
