<Update label="2026-04-18" description="Per-user config, press animations, and quality-of-life improvements">
## Per-user config, press animations, and quality-of-life improvements

A new `buttons config` command, smoother press feedback in the board, and several usability improvements across the CLI.

### New features

- **Per-user settings** — `buttons config set default-timeout 600` saves a default timeout that applies to every project on your machine. Settings live in `~/.buttons/settings.json` and are resolved in order: explicit `--timeout` flag first, then your config value, then the built-in 300-second default. Run `buttons config` with no arguments to see current values. See the [CLI reference](/cli/buttons_config).

- **Live elapsed counter** — buttons that are currently running now show a live clock in the board. Pinned cards display the elapsed time below the button name; list rows show it next to the "active" badge. Short presses read as `3.2s`, longer ones switch to `2:07` or `1:04:22`.

- **Press animation** — pressing a button in the board now plays a quick four-frame pulse that mimics a physical button press. The animation runs in under 200 milliseconds and gives immediate tactile feedback before the spinner takes over.

### Updates

- **"Next:" hints** — successful commands now print a suggested next step. For example, `buttons create` suggests `buttons press`, and `buttons press` suggests `buttons history`. Hints are suppressed in JSON mode and when piping output, so they never interfere with scripts.

- **Common flags in help text** — `buttons create --help` and `buttons press --help` now surface the most-used flags at the top of the output, before the full auto-generated list. No new flags — just easier discovery.

- **Automatic history pruning** — run history is now capped at the 100 most recent runs per button. Older runs are removed automatically after each press. This keeps the `pressed/` directory bounded without any manual cleanup.
</Update>

<Update label="2026-04-18" description="Logs pane in the board dashboard">
## Logs pane in the board dashboard

You can now see recent run history for any button without leaving the board.

### New features

- **Inline logs pane** — press `L` while focused on a button in the board to open a logs pane that shows the last five runs. Each row displays the result glyph, timestamp, exit code, duration, and a preview of the output. Press `L` again to hide it. See the [CLI reference](/cli/buttons_board).

- **At-a-glance results** — successful runs show a green check mark; failures show an orange cross. The output preview is the first line of stdout on success, or stderr on failure, truncated to fit your terminal width so every row stays on a single line.

- **Helpful empty states** — if no button is focused, the pane tells you to focus one. If the focused button has no runs yet, it prompts you to press it to record one.
</Update>

<Update label="2026-04-18" description="Higher default press timeout">
## Higher default press timeout

The default timeout for button presses has been raised from 60 seconds to 300 seconds.

### Bug fixes

- **Default timeout raised to 300s** — the previous 60-second default was too tight for realistic workloads like ETL jobs, migrations, and long-running API calls. If your first press on a real pipeline hit the timeout, it looked like the tool was broken. The new 300-second default gives scripts room to finish while still protecting against runaways. You can still shorten it per-button with `--timeout` on [create](/cli/buttons_create) or override per-press with `--timeout` on [press](/cli/buttons_press).
</Update>
