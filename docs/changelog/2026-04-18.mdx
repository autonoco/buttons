<Update label="2026-04-18" description="Per-user settings with buttons config">
## Per-user settings with `buttons config`

You can now save personal preferences that apply across all your projects.

### New features

- **`buttons config` command** — a new `buttons config` command lets you view, set, and clear personal preferences. Settings are stored per-user and never committed to project repos. Run `buttons config` to see current values, `buttons config set` to save a preference, and `buttons config unset` to clear one. Supports `--json` for machine-readable output.

- **Configurable default timeout** — the first available setting is `default-timeout`. Set it once with `buttons config set default-timeout 600` and every new button you create will use that timeout instead of the built-in 300 seconds. You can still override per-button with `--timeout` on [create](/cli/buttons_create) or per-press on [press](/cli/buttons_press).
</Update>

<Update label="2026-04-18" description="CLI quality-of-life improvements">
## CLI quality-of-life improvements

Small polish to make the CLI easier to learn and use day-to-day.

### Updates

- **"Next:" hints after commands** — after a successful command, the CLI now prints a suggested next step. For example, `buttons create` suggests `buttons press`, and `buttons press` suggests `buttons history`. Hints only appear in interactive terminals and are suppressed when using `--json` or piping output.

- **Common flags surfaced in help text** — `buttons create --help` and `buttons press --help` now show the most-used flags (like `--timeout`, `--arg`, and `--dry-run`) at the top, before the full flag list. No flags were added or removed — they're just easier to find. See the [create](/cli/buttons_create) and [press](/cli/buttons_press) CLI reference.

- **Automatic run history pruning** — run history in the `pressed/` directory is now automatically capped at 100 runs per button. Older runs are cleaned up after each press, so disk usage stays bounded without any manual maintenance.
</Update>

<Update label="2026-04-18" description="Logs pane in the board dashboard">
## Logs pane in the board dashboard

You can now see recent run history for any button without leaving the board.

### New features

- **Inline logs pane** — press `L` while focused on a button in the board to open a logs pane that shows the last five runs. Each row displays the result glyph, timestamp, exit code, duration, and a preview of the output. Press `L` again to hide it. See the [CLI reference](/cli/buttons_board).

- **At-a-glance results** — successful runs show a green check mark; failures show an orange cross. The output preview is the first line of stdout on success, or stderr on failure, truncated to fit your terminal width so every row stays on a single line.

- **Helpful empty states** — if no button is focused, the pane tells you to focus one. If the focused button has no runs yet, it prompts you to press it to record one.
</Update>

<Update label="2026-04-18" description="Higher default press timeout">
## Higher default press timeout

The default timeout for button presses has been raised from 60 seconds to 300 seconds.

### Bug fixes

- **Default timeout raised to 300s** — the previous 60-second default was too tight for realistic workloads like ETL jobs, migrations, and long-running API calls. If your first press on a real pipeline hit the timeout, it looked like the tool was broken. The new 300-second default gives scripts room to finish while still protecting against runaways. You can still shorten it per-button with `--timeout` on [create](/cli/buttons_create) or override per-press with `--timeout` on [press](/cli/buttons_press).
</Update>
